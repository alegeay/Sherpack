# Pack Structure

A Sherpack pack is a directory containing all the files needed to deploy an application.

## Directory Layout

```
mypack/
├── Pack.yaml             # Required: Pack metadata
├── values.yaml           # Required: Default values
├── values.schema.yaml    # Optional: JSON Schema for validation
├── Pack.lock.yaml        # Generated: Locked dependencies
├── packs/                # Downloaded dependencies
│   ├── redis/
│   └── postgresql/
└── templates/            # Required: Template files
    ├── deployment.yaml
    ├── service.yaml
    ├── configmap.yaml
    ├── ingress.yaml
    └── _helpers.tpl      # Optional: Shared helpers
```

## Required Files

### Pack.yaml

Metadata about your pack:

```yaml
apiVersion: sherpack/v1
kind: application
metadata:
  name: myapp
  version: 1.0.0
  description: My application
  appVersion: "2.0.0"
  maintainers:
    - name: Your Name
      email: you@example.com
  keywords:
    - web
    - nginx
```

### values.yaml

Default configuration values:

```yaml
app:
  name: myapp
  replicas: 1

image:
  repository: nginx
  tag: latest
```

### templates/

Directory containing Jinja2 template files. Each `.yaml` file is rendered and concatenated into the final manifest.

## Optional Files

### values.schema.yaml

JSON Schema for validating values:

```yaml
$schema: http://json-schema.org/draft-07/schema#
type: object
properties:
  app:
    type: object
    required: [name]
```

### Pack.lock.yaml

Generated by `sherpack dependency update`. Contains locked dependency versions:

```yaml
pack_yaml_digest: sha256:abc123...
policy: version
dependencies:
  - name: redis
    version: "7.0.0"
    repository: https://charts.example.com
    digest: sha256:def456...
```

### _helpers.tpl

Shared template macros that can be imported by other templates.

## Template Naming

- Files starting with `_` (like `_helpers.tpl`) are not rendered directly
- All other `.yaml` files are rendered and included in output
- Files are processed in alphabetical order
